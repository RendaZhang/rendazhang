# 前端目录所有权地图

- **作者**: 张人大
- **最后更新**: June 16, 2026, 22:13 (UTC+08:00)

## 文档目的

本文记录当前 Astro + React 前端代码树中各目录的职责边界，用于补充 [前端架构约定](./FRONTEND_ARCHITECTURE.md)。它描述当前代码已经采用的目录所有权，不记录假设中的目标架构。

本文是公开仓库安全文档，不包含密钥、生产凭据、内部主机信息、私有 IP 或一次性运维细节。

## 当前事实

- 当前项目以 Astro 页面、Astro layout、React islands、服务层、工具函数、内容常量和样式系统分层。
- 当前存在 `src/stores`，用于小型跨 island UI/preference 状态。
- 当前存在 `src/controllers`，用于不直接渲染 UI 的复杂交互编排边界。
- 当前不存在 `src/features` 目录。README 中的早期示意不应被理解为现有事实；新增文件应以本文件和实际代码树为准。
- 自动生成的 barrel 文件包含 `// AUTO-GENERATED by scripts/generateIndex.ts` 标记，不应手工编辑。

## 核心目录所有权

### `src/pages`

拥有：

- Astro 路由文件和页面入口。
- 路由级 metadata、canonical、Open Graph、Twitter Card 和 layout props。
- 页面静态 HTML 骨架。
- 页面级样式选择，包括 `extraStyles`、`preloadStyles` 和少量页面直接样式 import。
- 当前页面使用哪些 React island，以及对应 `client:*` hydration 指令。

不拥有：

- 可复用 UI 组件实现。
- API 请求细节。
- 全局 provider 状态。
- 通用样式 token。

当前导入方向：

- 可以导入 `src/layouts`、`src/components`、`src/content`、`src/constants`、`src/utils` 和必要的样式文件。
- 不应被 `src/components`、`src/services`、`src/utils` 或 `src/content` 反向导入。

### `src/layouts`

拥有：

- Astro layout shell，目前核心文件是 `BaseLayout.astro`。
- 文档结构、metadata、全局样式入口、顶层 slots 和顶层 islands。
- 主题、语言、标题和登录状态的渲染前初始化脚本挂载。

不拥有：

- 页面业务内容。
- React provider 具体实现。
- 具体 API 调用和表单流程。

当前导入方向：

- 可以导入顶层 React island、常量、工具函数和全局样式入口。
- 不应导入页面文件，也不应成为普通组件的横向依赖。

### `src/components`

拥有：

- React island 和可复用 React 组件。
- 当前交互 UI 的主要实现，包括导航、表单、页面 section、Chat、provider 和通用 UI。
- 组件级组合逻辑、React 生命周期、事件处理和局部展示状态。

不拥有：

- Astro 路由和文档 shell。
- 通用 API request 基础设施。
- 设计 token 定义。
- 纯内容数据的源头。

子目录职责：

| 子目录 | 所有权 |
| --- | --- |
| `layouts` | React 导航相关组件，包括 `NavBar`、`HamburgerMenu` 和 `NavBarWrapper`。`NavBarWrapper` 当前包裹 `ThemeProvider`、`I18nProvider`、`AuthProvider` 和 `ErrorBoundary`。 |
| `providers` | 当前主题、语言和认证 provider 状态。这里不是通用 store 替代品；跨 island UI/preference 读取通过 `src/stores` 的小型 store 边界处理。 |
| `sections` | 页面级内容组合组件，例如主页 sections、认证页 content wrapper、证书页内容、文档页效果和 profile 内容。 |
| `forms` | 表单 UI、验证、提交状态和表单级流程。认证表单使用 `apiClient`；`ContactForm` 当前保留直接提交联系端点的旧路径，不应作为新 JSON API 的默认模式。 |
| `chat` | Chat 页面、消息列表、输入框、Markdown 增强、流式会话 UI 和 `ChatWidget`。 |
| `ui` | 可复用 UI 原子或小组件，包括 display、inputs、media、widgets 等。 |
| 根级组件 | `ErrorBoundary`、`ErrorSection` 等跨页面复用组件。 |

### `src/components/chat`

拥有：

- `/deepseek_chat/` 页面主体和 Chat Widget 的 React UI。
- Chat 消息展示、输入、加载状态、typing 状态和 Markdown 增强触发。
- 同源 iframe ready 消息契约的 UI 侧处理。
- 当前 Chat 页面中的流式交互编排，直到后续 controller 切片拆分。

Chat Widget 与 `/deepseek_chat/` iframe 的消息类型、origin/source 校验和 ready 生命周期见：[Chat Widget 协议说明](./CHAT_WIDGET_PROTOCOL.md)。

不拥有：

- 普通 JSON API 的通用 request 行为。
- 非 Chat 功能的服务层。
- 全站 iframe/CSP 策略。
- 通用全局 store。

当前允许依赖：

- `src/services/chatService.ts` 的流式 Chat 请求。
- `src/hooks/useChatHistory.ts` 和 `src/hooks/useMarkdownPipeline.ts`。
- Chat 内容常量、Chat types、通用 UI、provider 语言状态和 Chat Widget 打开状态 store。

### `src/services`

拥有：

- 网络请求边界和后端 API 调用封装。
- `apiClient.ts` 的 typed JSON request、认证 cookie、JSON 解析、HTTP 错误和 401 处理。
- `chatService.ts` 的 Chat 流式 NDJSON 请求路径。

不拥有：

- React 组件、hooks 或 UI 状态。
- DOM 渲染、iframe 处理或页面跳转之外的展示逻辑。
- 文案内容。

当前导入方向：

- 可以导入 `src/constants`、`src/types` 和 `src/utils`。
- 不应导入 `src/components`、`src/pages`、`src/layouts` 或 `src/styles`。

### `src/utils`

拥有：

- 与 UI 无关的通用工具函数。
- 环境变量快照与别名解析。
- localStorage、sessionStorage、cookie、IndexedDB 和 memory fallback。
- 语言读取和默认语言 fallback。
- logger、密码规则和图片辅助。

不拥有：

- React 生命周期和组件状态。
- 网络请求 orchestration。
- 页面路由选择。
- 全局主题或认证 provider 状态。

工具函数可以被页面、组件、hooks 和 services 复用。新增工具应尽量保持纯函数或显式副作用；副作用范围应通过函数名和调用方清楚表达。

### `src/styles`

拥有：

- 全局 CSS 入口 `theme.css`。
- cascade layers：reset、tokens、base、components、utilities、overrides。
- 设计 token、渐变 token、layout base、组件样式和工具类。
- 样式说明文件，例如 `*.md` 和子目录 README。

不拥有：

- React 组件状态。
- 页面 metadata。
- API 或内容数据。

当前子目录职责：

| 子目录 | 所有权 |
| --- | --- |
| `core` | 基础颜色、间距、字体、阴影、Markdown/page 基础 token、主题语义别名、`data-theme` 覆盖和渐变 token。 |
| `components` | 导航、按钮、表单、modal、Chat、docs、certifications、profile 等组件或页面相关样式。 |
| `utilities` | 通用布局和工具类，例如 flex、grid、position、text helpers。 |

样式新增规则：

- 优先复用现有 token、layer 和类名前缀。
- `@media` 和 `@container` 查询条件使用可解析常量，不使用 CSS custom property。
- React 组件不要用大量内联样式绕过 token 系统；确实需要运行时尺寸时应保持局部。

### `src/content`

拥有：

- 中英文文案、页面内容常量、导航文案和认证错误文案。
- 可被 Astro 页面和 React 组件共享的静态内容数据。

不拥有：

- UI 渲染逻辑。
- API 请求。
- 浏览器存储、事件监听或 DOM 副作用。

当前允许依赖：

- 可以导入 `src/constants` 中的公共路径、图片、链接或站点常量。
- 不应导入 `src/components`、`src/services`、`src/hooks` 或 `src/pages`。

### `src/stores`

当前状态：存在。

当前拥有：

- 跨 island、跨页面或全局 UI 状态的最小共享边界。
- `uiPreferencesStore.ts` 的主题 mode、Chat Widget 打开/关闭状态和偏好 readiness。
- 可注入 storage adapter 的偏好持久化 helper。
- 供 React hook 和多个 island 读取的订阅/snapshot 接口。

不拥有：

- API 请求实现。
- React 组件渲染。
- Astro 页面 metadata。
- 后端业务规则。
- Chat 流式消息状态。
- 认证 session 状态。

修改条件：

- 只放小型客户端 UI/preference 状态。
- 保持 framework-neutral，不在 store 内导入 React。
- React 读取层放在 `src/hooks`，例如 `useUiPreferences.ts`。
- 新增状态库依赖前必须单独评审；当前没有使用 `nanostores`、Zustand 或 Jotai。

### `src/controllers`

当前状态：已存在。当前文件包括：

- `chatController.ts`：Chat 发送、stream chunk 回调、assistant 消息 upsert/finalize、reset 清理回调和 active request abort 边界。

拥有：

- 复杂交互流程的可测试 orchestration，例如 Chat controller。
- 连接 services、models、stores、hooks 回调和组件状态回调的业务流程边界。
- 不直接渲染 UI 的状态转换和流程控制。

不应拥有：

- React JSX。
- Astro 页面。
- CSS。
- 低层 fetch 细节。

修改条件：

- controller 可以接收组件传入的回调，但不应直接操作 DOM 或 iframe `postMessage` 协议。
- Chat 网络 transport、NDJSON 解析、401 处理和 reset endpoint 调用仍由 `src/services/chatService.ts` 拥有。
- 新增复杂流程前应先确认它不能更简单地保留在组件、hook、store 或 service 内。

## 支撑目录所有权

| 目录 | 所有权 |
| --- | --- |
| `src/constants` | 站点路径、API endpoints、metadata、存储键、UI 尺寸、timing 和资源路径常量。 |
| `src/types` | API、Chat、config 和全局 TypeScript 类型。 |
| `src/hooks` | 可复用 React hooks，连接 React 生命周期、浏览器 API、stores、models、utils 和组件。 |
| `src/models` | 纯领域模型，例如 `ChatSession`；不依赖 React、DOM 或网络。 |
| `src/data` | 非文案型静态数据，例如 hero 图片配置。 |
| `src/assets` | 静态资源、图片、PDF、音频、favicon 和 README 渲染副本。 |
| `src/scripts` | 构建或静态页面需要的脚本源码，例如 `base-layout-init.ts`。 |
| `src/__tests__` | Vitest 测试，覆盖 utils、services、hooks、forms 和 Chat 行为。 |

## 导入方向约定

推荐依赖方向：

```text
pages -> layouts -> components -> hooks -> models
pages -> components -> services -> utils
pages -> content/constants/utils/styles
components -> content/constants/hooks/services/stores/types/utils
hooks -> stores/models/types/utils
services -> constants/types/utils
stores -> constants
utils -> constants
styles -> styles only
```

禁止或需评审的方向：

- `services` 导入 `components`、`pages`、`layouts` 或 CSS。
- `stores` 导入 React 组件、Astro 页面、services 或 CSS。
- `utils` 导入 React 组件或页面。
- `content` 导入组件、hooks、services 或页面。
- `styles` 依赖 TypeScript 模块。
- 普通组件反向导入 `src/pages`。
- 手工编辑自动生成的 `index.ts` barrel 文件。

## 新增文件放置规则

- 新路由：放在 `src/pages`，只做路由级 metadata、layout props、静态结构和 island 选择。
- 新页面级交互：优先放在 `src/components/sections` 或对应 feature-like 子目录，而不是塞进 `.astro` 页面。
- 新通用 UI：放在 `src/components/ui` 的合适子目录。
- 新表单：放在 `src/components/forms/<FormName>`，认证相关 API 走 `apiClient`。
- 新 Chat UI：放在 `src/components/chat`；发送/reset 等复杂编排放在 `src/controllers/chatController.ts` 或同目录后续 controller；流式 transport 继续走 `src/services/chatService.ts`。
- 新普通 JSON API：放在 `src/services/apiClient.ts` 或同目录服务模块，并复用 `apiClient.request`。
- 新通用工具：放在 `src/utils`，避免隐式 UI 副作用。
- 新文案：放在 `src/content`，不要写进 service 或 utility。
- 新 token 或样式：放在 `src/styles` 对应 layer，并同步样式说明。
- 新跨 island UI/preference 状态：放在 `src/stores`，保持小型、客户端、framework-neutral；React 读取 helper 放在 `src/hooks`。
- 新复杂流程控制：放在 `src/controllers`，保持可测试、不渲染 UI、不拥有低层 fetch 细节。
